# PACS Python 环境构建子系统需求

## 目标

在 MiniOpenClaw 的现有 ReAct、工具注册和 Skill 加载框架上，实现并行自适应约束搜索（PACS），把 Python 环境配置从串行试错升级为隔离环境并行探测、失败约束提取和候选剪枝。

## 交付物

- `envpool`：隔离虚拟环境生命周期与并行 pip 安装。
- `resolver`：依赖解析、候选组合、失败日志结构化和 SQLite 约束图。
- 8 个返回字符串的 Agent 工具，并注册到默认工具表。
- PACS 评测任务和覆盖核心验收标准的自动化测试。

## 约束

- 仅使用 Python 标准库和项目已有依赖。
- 不侵入 ReAct 主循环；并行性由工具内部管理。
- 所有工具失败路径返回结构化字符串，不向 Agent 泄漏未处理异常。
- 持久化数据默认位于项目 `.miniopenclaw/`，可由环境变量覆盖。
- 不执行真实大型第三方包下载作为自动化测试的一部分。

## 验收标准

以 `.plans/006-pacs-env-builder/done-standard.md` 的 F1-F8 和非功能性条目为准。自动化验证至少覆盖：环境创建/状态/清理、并行任务隔离、requirements 与 pyproject 解析、候选限制与冲突剪枝、15+ 失败模式、传递约束持久化、8 工具注册、Skill 加载及模块导入。

## 非目标

- 修改模型 ReAct 调度算法。
- 实现完整 PyPI 版本索引客户端或通用 SAT 求解器。
- 在测试中安装 torch、numpy 等大型包。

## 交付真实性

只有在自动化测试和导入检查均通过、工作区无临时运行残留时，才使用“完成”措辞；网络依赖的真实项目安装只声明接口和轻量验证结果。
