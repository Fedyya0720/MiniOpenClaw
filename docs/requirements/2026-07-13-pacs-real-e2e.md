# PACS 真实端到端增量需求

## 目标

把 PACS 从组件 MVP 补齐为可直接运行的环境构建流程，并用真实 pip 依赖解析完成验收。

## 必须交付

- 从 PyPI JSON API 或注入的版本目录发现真实发布版本。
- 依据项目约束、历史冲突和已尝试集合生成实际存在的候选。
- 多环境真实并行 pip 安装；失败日志进入约束图并驱动后续候选。
- 成功环境执行 import 验证，生成 `requirements.lock` 和 `PACS_REPORT.md`。
- 自动清理失败环境，保留成功环境。
- 提供 Python API、`python -m pacs.cli` 与 Agent 高层工具 `pacs_build`。

## 产品验收

受控真实项目必须至少包含一个不可安装候选和一个可安装候选；运行中必须观察到真实 pip 失败后继续搜索并成功。最终锁文件可复现，报告记录尝试、失败、成功、环境路径和约束数据库。

## 约束

- 生产代码仅使用标准库。
- 版本目录可注入，以支持确定性离线测试；真实验收使用 PyPI 和真实 pip。
- 不把启发式猜测版本当作“真实候选”。
- 不覆盖源 `requirements.txt`；锁文件单独写出。

## 完成语言

只有真实 pip E2E、自动化测试、全量回归和产物核查全部通过，才能声明端到端能力完成。
