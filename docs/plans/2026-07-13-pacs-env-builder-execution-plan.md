# PACS 执行计划

## 执行等级

L：单代理、分阶段串行实施。模块边界独立，但注册表与端到端测试共享接口，串行执行更利于保持契约一致。

## 阶段

1. 基线与治理：冻结需求、记录 skeleton 和执行范围。
2. 环境池：实现数据模型、创建/查询/清理、并行子进程安装与 4 个工具。
3. 解析器：实现依赖声明解析、候选生成、失败规则、SQLite 约束图与 4 个工具。
4. 集成：默认注册表、权限提示、评测任务和测试。
5. 验证清理：运行 `python -m unittest discover -s tests -v`、导入检查、状态审计并清理测试产物。

## 所有权边界

- 新代码限于 `envpool/`、`resolver/`、PACS 工具、测试和计划要求的集成文件。
- 不恢复或混入本地 stash 中的旧 Python environment agent MVP。

## 回滚规则

每阶段保持模块可导入；若接口设计与现有 Tool 契约冲突，回退该阶段局部实现，不重写现有框架。

## 完成语言

测试通过后报告已验证条目；未执行的大型联网安装和真实模型端到端调用明确列为未验证。
