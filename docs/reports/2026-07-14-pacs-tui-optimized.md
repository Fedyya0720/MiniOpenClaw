# PACS 优化后 TUI 复测简报

日期：2026-07-14  
环境：macOS，Python 3.14.4，`deepseek-v4-flash`  
方法：5 个项目，每项使用两个全新副本并发执行真实 TUI PACS，共 10 次；计时从提交自然语言请求到 TUI 返回输入提示符。

## 本轮优化

- PyPI JSON 版本发现改为最多 8 路并发，并对瞬时网络错误执行退避重试；
- preflight 与候选环境共享项目级 pip 下载缓存；
- 默认尝试数从 6 恢复到 8；
- 成功候选继续 editable 安装项目本体，再执行 `pip check`；
- macOS 不再应用按整个登录用户计数的 `RLIMIT_NPROC=128`，避免 pip 调用 git 时出现 `EAGAIN`；Linux Docker 仍保留进程数限制。

## 结果

| 项目 | TUI 两次 / s | TUI 均值 / s | PACS 内部均值 / s | 成功 | 优化前 TUI 均值 / s |
|---|---:|---:|---:|---:|---:|
| sampleproject | 24.89 / 39.21 | 32.05 | 16.09 | 2/2 | 30.34 |
| uv-fastapi-example | 76.95 / 78.66 | 77.81 | 69.44 | 2/2 | 81.61 |
| requests | 68.76 / 41.51 | 55.13 | 46.53 | 2/2 | 28.87 |
| flask | 39.20 / 39.66 | 39.43 | 29.66 | 2/2 | 28.92 |
| poetry | 134.41 / 170.03 | 152.22 | 142.52 | 2/2 | 598.09 |

汇总：

- PACS 成功率由 **8/10 提升到 10/10**；
- 10 个 winner 均完成 editable 安装、`pip check`，并通过项目主模块独立 import；
- 五项目等权端到端均值由 **153.57 s 降到 71.33 s**，耗时降低约 **53.6%**，整体约快 **2.15×**；
- Poetry 均值由 **598.09 s 降到 152.22 s**，约快 **3.93×**，且不再触发 Agent 手工降级；
- 单独运行 Poetry PACS 的冷启动为 **50.63 s**；双重复并发时 Git/PyPI 下载争用使其升到 124～161 s，说明并发基准会放大外部网络竞争；
- 小项目端到端时间没有全面下降，因为本轮新增了项目本体 editable 安装，而且模型/API 与网络方差仍占明显比例。Requests 的一次慢运行是主要离群点。

## 结论

本轮首先修复了正确性和大型项目退化：Poetry 已恢复到可稳定完成，且单独 PACS 时间与参考分支的 58.15 s 相当。下一步若继续追求小项目速度，应减少重复 pip preflight/安装工作，并将模型最终汇报压缩为一次调用；不能通过取消项目安装和验证来制造更快但不完整的结果。
