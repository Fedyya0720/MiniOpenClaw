Day 8 下午 · 规划层：TodoList + todo 驱动 + 反思 + 错误恢复 + 有界停止

一句话目标：为 agent 加一层审议式规划——让它把长任务拆成清单、逐条推进、出错能恢复、知道何时停。 本次产出：一个 TodoList（分解 + 状态机）+ todo 工具、把 todo 注入主循环、一个反思 / 重试步骤、失败恢复（瞬时重试 / blocked）、无进展检测与有界停止。 涉及的 starter 文件：新建 agent/planning.py（TodoList）；工具注册在 tools/；注入与调度在 agent/loop.py。（本课新增内容，starter 暂无 TODO[Day8] 桩，按下面步骤新建即可。）

开始之前

  • 所有命令都在 starter/ 目录下运行：cd starter。
  • 承接前几天：agent 已有主循环、工具集、MCP、Skills、安全层、记忆。
  • 先想清楚边界（讲义 §2）：今天不改主循环本身（那是执行引擎），也不做多智能体——只在其上加一层规划。

步骤 1 · TodoList：分解 + 状态机（新建 agent/planning.py）

  1. 治迷失的核心（讲义 §4）：把大目标拆成有序、可跟踪的子任务，并把清单常驻上下文。 
    class TodoList:
    def __init__(self):
        self.items = []

    def write(self, texts):                       # 一次性写下分解后的清单
        self.items = [{"id": i + 1, "text": t, "status": "pending"}
                      for i, t in enumerate(texts)]

    def update(self, id, status):                 # pending/in_progress/completed/blocked
        for it in self.items:
            if it["id"] == id:
                it["status"] = status

    def insert(self, text):                        # 重规划时插入新子任务
        self.items.append({"id": len(self.items) + 1, "text": text, "status": "pending"})

    def render(self):                              # 注入上下文的样子（模型每轮都看得见）
        mark = {"pending": "[ ]", "in_progress": "[~]", "completed": "[x]", "blocked": "[!]"}
        return "\n".join("%s %d %s" % (mark[i["status"]], i["id"], i["text"]) for i in self.items)

    def all_done(self):
        return all(i["status"] == "completed" for i in self.items)
  2. ✅ 验证： 
    python -c "from agent.planning import TodoList; t=TodoList(); t.write(['A','B']); t.update(1,'completed'); print(t.render())"

步骤 2 · todo 工具：让模型自己列清单、勾进度（文件：tools/）

  1. 把 TodoList 暴露成工具，模型开始复杂任务时先 todo_write、每步 update_todo： 
    TODO = TodoList()   # 单会话内的规划状态
todo_write_tool = Tool(
    name="todo_write",
    description="面对多步任务时，先把它分解成有序子任务清单。传入子任务文本数组。",
    parameters={"type": "object", "properties": {
        "items": {"type": "array", "items": {"type": "string"}}}, "required": ["items"]},
    run=lambda items: (TODO.write(items), TODO.render())[1])
update_todo_tool = Tool(
    name="update_todo",
    description="完成或开始某条子任务时更新其状态。",
    parameters={"type": "object", "properties": {
        "id": {"type": "integer"},
        "status": {"type": "string", "enum": ["in_progress", "completed", "blocked"]}},
        "required": ["id", "status"]},
    run=lambda id, status: (TODO.update(id, status), TODO.render())[1])
    注册进 registry。这正是 Claude Code 的 TodoWrite。
  2. ✅ 验证：给一个多步任务，观察模型是否先调 todo_write 再逐条 update_todo： 
    python -m agent.cli "把项目里所有 requests 调用迁移到 httpx 并跑测试"

步骤 3 · 把 todo 注入主循环：让全局始终可见（文件：agent/loop.py）

  1. 防漂移的根本手段（讲义 §8.3）：每轮把当前 todo 状态拼进上下文，模型每步都知道"整体到哪了、下一条做什么"： 
    # 主循环里，每轮构造消息时
if TODO.items:
    messages.append({"role": "system",
                     "content": "# 当前任务清单（推进它，别跑偏）\n" + TODO.render()})
  2. ✅ 验证：长任务执行中途打印上下文，确认每轮都带着最新清单；模型不再重复已完成项、不漏未完成项（对比讲义 §1 的反应式）。

步骤 4 · 反思与错误恢复（文件：agent/loop.py / agent/planning.py）

  1. 反思（讲义 §6）：在子任务完成后 / 命令失败后插一步自我审视，发现问题就修正——并设上限（同一子任务最多反思 N 次），避免无限套娃。
  2. 错误恢复（讲义 §7）：分类失败—— 
    def with_retry(fn, max_tries=3, base=0.5):
    for k in range(max_tries):
        try:
            return fn()
        except TransientError:              # 瞬时：重试 + 指数退避
            time.sleep(base * 2 ** k)
    return None                             # 超限 → 交给 blocked 逻辑
# 永久失败 → 重规划（改 todo）；反复失败 → TODO.update(id, "blocked")，先做别的
  3. ✅ 验证：构造一个会失败的子任务（如故意让某处测试挂），确认 agent 不崩、把它标 blocked 或重试后修复，其它子任务照常推进（单步失败 ≠ 整体失败）。

步骤 5 · 长程控制：无进展检测 + 有界停止（文件：agent/loop.py）

  1. 无进展 / 绕圈检测（讲义 §8.2）：记录最近动作，连续 N 步无 todo 推进、或同一 (工具+参数) 反复出现 → 强制重规划 / 求助。
  2. 完成判据 + 预算（讲义 §8.1/§8.3）：TODO.all_done() 且验证通过才算完成；设步数上限（如 40 步），到顶停下汇报现状，别无声地烧。 
    if step > MAX_STEPS:
    return "已达步数上限，当前进度：\n" + TODO.render()
if TODO.all_done() and verify_passed():
    return "任务完成。"
  3. ✅ 验证：跑一个较长任务，确认它能正常终止（不早停、不空转），并在卡死时被 loop detection 打断。

本日构建目标（不单独计分，计入最终展示）

  • ☐ TodoList（分解 + 状态机）+ todo 工具可用，模型能列清单、勾进度。
  • ☐ todo 每轮注入主循环，全局目标常驻——长任务不漏不重（对比反应式）。
  • ☐ 反思 / 重试步骤，且有上限（不无限套娃）。
  • ☐ 错误恢复：瞬时重试退避 / 卡住 blocked，单步失败不拖垮整体。
  • ☐ 无进展检测 + 完成判据 + 步数预算，任务能正常终止、卡死能打断。

常见报错

  • 现象：模型不列 todo 直接猛干 → todo_write 的 description 没说清"何时用" → 写明"面对多步任务时先分解"。
  • 现象：模型列了 todo 但从不 update → 提示词没要求每步更新，或 todo 没注入上下文 → 步骤 3 每轮注入 + 系统提示要求勾进度。
  • 现象：一个子任务失败，整个 agent 崩了 → 没做失败分类 → 加 with_retry + blocked，把"单步"与"整体"分开。
  • 现象：agent 原地打转烧 token → 没有无进展检测 / 没有步数上限 → 步骤 5 两者都加。
  • 现象：反思停不下来 → 没设上限 → 同一子任务反思/重试超 N 次即 blocked 或求助。
