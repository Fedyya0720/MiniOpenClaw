Day 10 · Demo Day（最终展示，95%）

课程最后一天。每组 15–20 分钟：现场演示（含当场随机任务）+ 实现讲解 + 答辩，并提交源码 / 消融 / 文档。 这是全课唯一的大考——占总分 95%。九天亲手搭起来的 mini-OpenClaw，今天现场验收。 完整评分细则（8 类打分点，合计 95 分）见 ../00-课程总览/考核与评分总表.md §2。

1. 流程

  • 每组 15–20 分钟：约 5 分钟讲架构与设计亮点 → 8–10 分钟现场任务（含随机任务）→ 答辩。
  • 架构讲解要点明用上了哪些层：backend / 主循环 / 工具 / 上下文 / MCP / Skill / 安全 / 记忆（D7）/ 规划（D8）/ 可观测（D9）。
  • 现场随机任务：评委当场给一个未提前透露的任务，偏向你组的领域方向。
  • 组内每位成员都要参与讲解 / 答辩——评分含"组内每人都能回答机理问题"，别只让一个人上。

2. 现场任务是什么样的

现场任务通常满足： - 需要多工具协同（不是一招能完成）； - 难度适中：一个健壮的 agent 约 5 分钟内可完成； - 尽量用得上你组的扩展能力（MCP 工具 / 你写的 Skill）； - 可能是一个较长的多步任务（10+ 步），考察你的规划 / todo（D8）； - 可能故意制造一次失败（比如改个路径让某工具失败），考察你的错误恢复； - 可能跨"两次会话"，考察你的跨会话记忆（D7）； - 可能要求你用 trace / token 账说明这次运行怎么跑的、花了多少，考察可观测性（D9）。

示例任务（按领域方向，仅供感受难度）： - 数据分析：读 data/sales.csv，找出销售额最高的 3 个月，生成 report.md。 - 日志排障：data/app.log 里有一类反复出现的 ERROR，定位根因并给修复建议。 - 代码库导读：给一个陌生小 repo，生成模块说明 + 列出所有 TODO（多步，观察它怎么列 todo、逐条推进）。 - 科研辅助：读 paper.txt，提取研究问题 / 方法 / 主要结论，结构化输出。

能力变体（评委可能叠加）： - 记忆变体：先在一次会话里告诉 agent 一个项目约定，新开会话再验证它是否记得并遵循。 - 规划变体：一个 10+ 步的迁移 / 重构任务，看它是否列 todo、逐条推进、不漏不重、能从失败中恢复。 - 可观测变体：任务完成后，请你回放 trace，说明它走了哪几步、哪一步最贵（token）。

3. 评分：8 类打分点（合计 95 分，细则见考核总表 §2）

打分点 分 现场怎么看
A 系统完整性与可运行性 18 能一键启动；各层齐全：backend / 主循环 / 工具 / MCP / Skills / 安全 / 记忆 / 规划 / 可观测
B 现场任务完成度 18 当场随机任务能自主完成且结果正确；多工具协同
C 主循环与规划 14 多轮调用、结果回填、正确终止、无死循环；长任务能列 todo、逐条推进（D8）
D 扩展能力（MCP + Skills） 12 接真实 MCP server；领域 Skill 被召回并执行
E 上下文 / 记忆与鲁棒性 10 compaction 不撑爆；跨会话记忆生效（D7）；工具失败能自愈不崩
F 安全机制 8 权限分层 / 危险命令拦截；注入防护 + 一次红队演示
G 技术理解与答辩 9 讲清如何实现与为何这样设计；能用 trace / 成本佐证（D9）；组内每人能答
H 消融实验与技术文档 6 ≥1 组消融有数据有结论；架构文档（含安全 + 可观测小节）+ 各模块 README

4. 提交物（随展示）

  • git 仓库（含 v1 / v3 / final tag，python -m agent.cli "任务" 可运行）
  • 消融实验报告（用数据证明某个设计确实有用）
  • 技术文档（含安全小节与可观测 / 成本小节）

5. 给参演小组的建议

  • 开场先自检：现场先跑 python -m agent.cli --selfcheck，让评委看到系统是活的。
  • 主动展示恢复力：允许 agent 现场出错——准备一个"故意失败"的桥段，展示它怎么从错误里恢复。往往比一次成功更能体现功力。
  • 用 trace / 成本说话（D9）：讲解时打开一次运行的 trace，指着它说"这一步最贵、这里我做了并行 / 缓存优化"，比空谈设计更有说服力。
  • 每人讲自己负责的那层：答辩按人分工，主循环 / 工具 / MCP / Skill / 安全 / 记忆 / 规划 / 可观测各有人能答机理。
  • 消融要有数据：有 / 无 compaction、有 / 无 todo 规划、有 / 无错误恢复、有 / 无记忆——各跑一组对比，用数字说话。

一句话：今天不是背 PPT，而是让你亲手搭的 agent 当场干活、当场出错又当场恢复，并讲清它每一层为什么这么设计。
