Day 5 下午 · Skills 加载器 + 领域 Skill + 图像通道 + v3

一句话目标：实现 Skills 加载器、为你组领域写一个真能用的 Skill、给 agent 接上图像输入通道，并把 MCP + Skills + 多模态 + 完整工具集合成 v3。 本次产出：load_skills/skills_catalog 可用、catalog 进 system prompt、一个领域定制 SKILL.md 能被召回；agent 能把一张图编码为内容块送入模型（多模态通道打通）；打 tag v3。 涉及的 starter 文件与 TODO：skills/loader.py 的 TODO[Day5]（parse_skill_md）；参考 skills/example-skill/SKILL.md；注入点在 agent/prompts.py / agent/cli.py；图像通道在 backend/（消息构造）。

开始之前

  • 所有命令都在 starter/ 目录下运行：cd starter。
  • 承接 Day4：MCP 工具已可透明并入 registry。
  • 定位今天要填的地方： 
    grep -rn "TODO\[Day5\]" .
    应落在 skills/loader.py。先读 skills/example-skill/SKILL.md，看清 frontmatter（--- 包裹的 name/description）+ 正文 body 的约定。

步骤 1 · 实现加载器（文件：skills/loader.py）

  1. 打开 skills/loader.py，找到 parse_skill_md 的 TODO[Day5]。load_skills（扫描 skills/*/SKILL.md）与 skills_catalog（渲染 name+description 清单）已给出可用实现，parse_skill_md 是核心。
  2. 实现 parse_skill_md(text, path)——解析 --- frontmatter 的 yaml + 正文 body（骨架给全）： 
    def parse_skill_md(text: str, path: Path) -> Skill:
    import yaml
    name = description = ""
    body = text
    if text.startswith("---"):
        _, fm, body = text.split("---", 2)   # 头尾两个 --- 之间是 frontmatter
        meta = yaml.safe_load(fm) or {}
        name = meta.get("name", "")
        description = meta.get("description", "")
    return Skill(name=name, description=description, body=body.strip(), path=path)
    无 yaml 依赖时可手写：逐行找 name: / description:，第二个 --- 之后是 body。
  3. ✅ 验证： 
    python -c "from skills.loader import load_skills, skills_catalog; print(skills_catalog(load_skills()))"
    期望：打印一行 - csv-quick-report: 当用户给一个 CSV 文件...（示例 skill 被解析出 name + description）。

步骤 2 · 把 catalog 注入 system prompt（文件：agent/cli.py 或 agent/prompts.py）

  1. 让模型知道有哪些 skill：把 skills_catalog(load_skills()) 拼进 system prompt。在 agent/cli.py 的 main() 里构造 agent 前： 
    from skills.loader import load_skills, skills_catalog
skills = load_skills()
system = SYSTEM_PROMPT + "\n\n# 可用 Skills（相关时按其流程执行）\n" + skills_catalog(skills)
agent = AgentLoop(backend, reg, system)
  2. 简单版可把全部 skill 的 body 都拼进去；进阶做按需召回：任务命中某 skill 的 description 关键词时，才把该 skill 的 body 注入（呼应 Day4 上下文管理，避免 skill 多时占满上下文）。
  3. ✅ 验证： 
    python -c "from agent.prompts import SYSTEM_PROMPT; from skills.loader import load_skills, skills_catalog; \
print(SYSTEM_PROMPT + chr(10) + skills_catalog(load_skills()))" | tail -5
    期望：输出末尾能看到 skill 目录行，说明 catalog 已并进提示词。

步骤 3 · 写你组领域的 Skill（文件：skills/<your-skill>/SKILL.md）

  1. 按 Day1 选的方向，新建目录与文件 skills/<your-skill>/SKILL.md（不要只改示例的名字）。frontmatter 两个字段： - name：短横线命名，如 paper-digest。 - description：一句话说清「何时该用」——这是召回的关键。
  2. 正文写可执行的分步流程：用哪些工具、什么顺序、注意事项、示例输出。可选附 scripts/ 脚本并在正文引用。领域参考： - 科研辅助 → 论文速读报告（读 PDF/文本 → 提取贡献/方法/结论 → 出结构化要点） - 数据分析 → 扩展 csv 报告（按你们要的指标/图表） - 日志排障 → 异常栈定位（grep 报错 → 定位文件 → 给修复建议） - 代码库导读 → 陌生仓库导读（glob/grep 摸结构 → 出模块说明）
  3. ✅ 验证： 
    python -c "from skills.loader import load_skills; \
print([(s.name, s.description[:20]) for s in load_skills()])"
    期望：列表里出现你新建的 skill（name + description）。再给一个该领域任务跑 python -m agent.cli "<任务>"，确认模型召回并按 Skill 流程推进；对比「没有这个 Skill」时的表现可作一组消融。

步骤 4 · 整合 v3 并打 tag（MCP + Skills + 完整工具集）

  1. 确认三者能在同一个 agent 里协同：内置工具集（Day4）、MCP 工具（Day4）、Skills（今天）都在 agent/cli.py 的启动路径里接上。
  2. 跑一个综合任务：用到至少一个 MCP 工具 + 触发你的 Skill。 
    python -m agent.cli "<一个既需要 mcp__ 工具、又匹配你 Skill 的领域任务>"
  3. ✅ 验证：观察输出里既有 mcp__ 前缀工具被调用，也走了你 Skill 的流程步骤，任务正确完成。
  4. 打 tag： 
    git add -A && git commit -m "day5: v3 — MCP + Skills + 多模态图像输入"
git tag v3

步骤 5 · 给 agent 接上图像输入通道（多模态，文件：backend/）

目标：让 agent 的输入从纯文本扩展到「文本 + 图像」。核心只有一件事——把一张图编码成内容块，拼进 user 消息。模型须支持视觉；若当前 DeepSeek 模型暂不支持视觉输入，用一个支持视觉的模型验证该通道，重点是打通结构。

  1. 写一个把图片编码为内容块的小工具（放 backend/ 或 tools/）： 
    import base64
def image_block(path, media_type="image/png"):
    b64 = base64.b64encode(open(path, "rb").read()).decode()
    # Anthropic 风格内容块（本课端点）
    return {"type": "image", "source":
            {"type": "base64", "media_type": media_type, "data": b64}}
    大图先缩放（长边 ≤1568px，见讲义 §10.3）再编码，省 token、避免被拒。
  2. 让消息的 content 支持「内容块列表」：构造一条 user 消息，把文本块 + image_block(...) 混排送给后端。 
    msg = {"role": "user", "content": [
    {"type": "text", "text": "看下这张截图，描述你看到了什么"},
    image_block("demo/sample.png")]}
  3. ✅ 验证：给一张能看清内容的图（如一段代码截图 / 一张图表），让模型描述图里内容；模型的回答与图实际内容吻合，即通道打通。 
    python -m agent.cli --image demo/sample.png "这张图里显示了什么？"
    （若走工具返回图像的路径：写一个返回 image_block 的 screenshot/渲染工具，让模型看到工具返回的图后继续推理——即讲义 §11.2 的闭环。）
  4. （可选，加分）把多模态与 Skill 结合：写一个「看截图排 UI bug」类 Skill（讲义 §12.1），description 命中「提供截图 + 显示异常」，步骤第 1 步即「看图定位」。

常见报错

  • 现象：frontmatter 解析失败 → --- 边界或 yaml 缩进不对 → 用 yaml.safe_load，或手写逐行解析并确认头尾两个 ---。
  • 现象：ModuleNotFoundError: yaml → 依赖未装 → pip install pyyaml，或改用手写解析。
  • 现象：Skill 没被召回 → description 太泛或与任务无关，或 catalog 没进 system prompt → 收紧 description 的「何时用」，确认步骤 2 的注入生效。
  • 现象：Skill 和示例没区别 → 必须是你领域定制的真实流程 → 重写正文为你方向的分步操作。
  • 现象：skill 多时上下文被占满 → 全部 body 都注入了 → 改按需召回，只注入命中任务的那个 body。
