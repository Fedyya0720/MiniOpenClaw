g07 测试说明（学生版）—— Python 环境配置智能体（PACS）

打开 http://10.130.130.9:8007 的终端，入口 python -m agent.cli "..."（以你们 README 为准）。需要 DEEPSEEK_API_KEY，会建很多 venv、留足磁盘。

评委会考：给装不上的项目配出能用的环境，还要比一个个试更快。

A 你现场演示 + 答辩

  • 现场配一个依赖打架的项目，展示它一次试好几个版本组合、发现不兼容就整片跳过。
  • 答辩：讲一次失败怎么变成「这两个版本别再试了」、并行是怎么调度的。

B 评委当场用命令考你

这些命令你先自己都跑通；命令里 <...> 的素材（网页、仓库、视频号之类）自己先准备好。

B1 干净项目

  • 命令：time python -m agent.cli "给这个项目配好环境：<一个依赖简单的项目>"
  • 过：配出来能 import、能跑；记一下用了多久
  • 没过：配不出/装完用不了

B2 冲突项目

  • 命令：time python -m agent.cli "给这个项目配好环境：<一个 torch 和 numpy 版本打架的项目>"
  • 过：把「这俩版本不兼容」记下来、跳过一整片、最后还是配出能用的
  • 没过：一个个死试、超时或放弃

B3 比快慢

  • 命令：# 同一个冲突项目，串行试 和 你们并行搜 各跑一次比 time
  • 过：并行明显快、少试很多次（可直接写进消融）
  • 没过：并行没优势，或根本没并行

B4 装坏包

  • 命令：python -m agent.cli "配环境：<requirements 里放一个装的时候会跑 rm 的包>"
  • 过：装包在沙箱里、有资源上限，没让它乱来
  • 没过：恶意包被无脑装/执行

C 评委翻你的代码核实

在你们容器里跑（自己先跑一遍看看）：

C1 grep -rniE "ThreadPool|Process|Pool|并行|parallel|async" tools/ | head

  • 应看到：真是并发跑的
  • 要是「只是串行 for 循环，快不起来」，说明没真做。

C2 grep -rniE "constraint|约束|prune|剪枝|conflict" . | head

  • 应看到：有约束/剪枝逻辑
  • 要是「没有，纯靠试」，说明没真做。

C3 grep -rniE "parse.*error|结构化|structured|traceback" tools/ | head

  • 应看到：失败日志被结构化解析、复用
  • 要是「只把原始报错丢给模型」，说明没真做。
