步骤 1 · 实现 read / write（文件：tools/fs.py）

  1. 打开 tools/fs.py，找到 _read 与 _write 里的 TODO[Day4]。
  2. 实现 _read(path, max_bytes=100_000)：读文本、超长截断并提示、带行号（利于后续 edit 定位原文）。骨架： 
    def _read(path: str, max_bytes: int = 100_000) -> str:
    with open(path, "r", encoding="utf-8") as f:
        text = f.read(max_bytes + 1)
    truncated = len(text) > max_bytes
    if truncated:
        text = text[:max_bytes]
    lines = text.splitlines()
    body = "\n".join(f"{i:>6}\t{ln}" for i, ln in enumerate(lines, 1))
    if truncated:
        body += f"\n... [已截断，仅显示前 {max_bytes} 字节]"
    return body or "[空文件]"
  3. 实现 _write(path, content)：覆盖写入，返回含字节数与路径的成功提示。骨架： 
    def _write(path: str, content: str) -> str:
    with open(path, "w", encoding="utf-8") as f:
        n = f.write(content)
    return f"已写入 {n} 字节到 {path}"
  4. ✅ 验证：

步骤 2 · 实现 bash（文件：tools/shell.py）

  1. 打开 tools/shell.py，找到 _bash 里的 TODO[Day4]。
  2. 实现 _bash(command, timeout=30)：用 subprocess.run 执行，捕获 stdout/stderr/returncode，带超时。必须捕获 stderr 与非零 returncode 并原样喂回——失败信息要「响亮」，模型才能自我修复。骨架： 
    import subprocess

def _bash(command: str, timeout: int = 30) -> str:
    try:
        p = subprocess.run(
            command, shell=True, capture_output=True, text=True, timeout=timeout,
        )
    except subprocess.TimeoutExpired:
        return f"[超时] 命令超过 {timeout}s 未结束：{command}"
    out = p.stdout or ""
    if p.stderr:
        out += f"\n[stderr]\n{p.stderr}"
    if p.returncode != 0:
        out += f"\n[returncode={p.returncode}]"
    return out.strip() or "[无输出]"
    先不做沙箱（Day 6 加权限层），但超时和 stderr 捕获现在就要有。
  3. ✅ 验证： 
    python -c "from tools.shell import _bash; print(_bash('echo hi && ls nope_dir'))"
    期望打印 hi，并带上 [stderr] ...No such file... 和 [returncode=...]。

步骤 4 · 实现 edit（用 diff 写文件，文件：tools/more_tools.py）

  1. 打开 tools/more_tools.py，找到 _edit 里的 TODO[Day4]。
  2. 实现 _edit(path, old="", new="")，用最稳的 search-replace（不依赖行号的 diff）：读入全文，统计 old 出现次数；唯一（恰好 1 次）才替换并写回；0 次或多次都返回清晰失败信息，提示模型扩大 old 范围让它唯一。骨架： 
    def _edit(path: str, old: str = "", new: str = "") -> str:
    with open(path, "r", encoding="utf-8") as f:
        text = f.read()
    count = text.count(old)
    if count == 0:
        return f"[失败] 未找到待替换文本，请照抄文件原文（含缩进）。path={path}"
    if count > 1:
        return f"[失败] old 在文件中出现 {count} 次，不唯一；请扩大 old 片段使其唯一。"
    with open(path, "w", encoding="utf-8") as f:
        f.write(text.replace(old, new, 1))
    return f"已在 {path} 完成 1 处替换。"
  3. ✅ 验证：

步骤 5 · 实现 grep（文件：tools/more_tools.py）

  1. 找到 _grep 里的 TODO[Day4]。
  2. 实现 _grep(pattern, path=".")：调系统 rg，输出 文件:行号:内容，结果过多时截断并提示。骨架： 
    import subprocess

def _grep(pattern: str, path: str = ".", max_lines: int = 100) -> str:
    try:
        p = subprocess.run(
            ["rg", "--line-number", "--no-heading", pattern, path],
            capture_output=True, text=True, timeout=30,
        )
    except FileNotFoundError:
        return "[失败] 未找到 rg，请先安装 ripgrep。"
    if p.returncode not in (0, 1):  # 1 = 无匹配，属正常
        return f"[grep 出错] {p.stderr.strip()}"
    lines = p.stdout.splitlines()
    if not lines:
        return f"[无匹配] pattern={pattern}"
    if len(lines) > max_lines:
        return "\n".join(lines[:max_lines]) + f"\n... [共 {len(lines)} 行，已截断前 {max_lines} 行]"
    return "\n".join(lines)
  3. ✅ 验证：

步骤 6 · 实现 glob（文件：tools/more_tools.py）

  2. 实现 _glob(pattern)：用 pathlib 的 rglob 递归匹配文件名，返回路径列表，过多截断。骨架： 
    from pathlib import Path

def _glob(pattern: str, max_items: int = 100) -> str:
    paths = [str(p) for p in Path(".").rglob(pattern) if p.is_file()]
    if not paths:
        return f"[无匹配] pattern={pattern}"
    if len(paths) > max_items:
        return "\n".join(paths[:max_items]) + f"\n... [共 {len(paths)} 个，已截断前 {max_items} 个]"
    return "\n".join(paths)

步骤 12 · 实现 start 与 _rpc（文件：mcp/client.py）

  1. 打开 mcp/client.py，找到 MCPClient.start 的 TODO[Day4]。
  2. 实现 start：起 stdio 子进程并做 initialize 握手： 
    def start(self) -> None:
    self.proc = subprocess.Popen(
        self.command,
        stdin=subprocess.PIPE, stdout=subprocess.PIPE,
        text=True, bufsize=1,          # 行缓冲，配合一行一条消息
    )
    self._rpc("initialize", {
        "protocolVersion": "2024-11-05",
        "capabilities": {},
        "clientInfo": {"name": "mini-openclaw", "version": "0.1"},
    })
    self._notify("notifications/initialized")   # 通知，无需等 result
  3. 实现 _rpc(method, params)（TODO[Day4]）：写一行 JSON + flush，读一行响应，按 id 取 result。再补一个 _notify（发通知，不读响应）： 
    def _rpc(self, method: str, params: dict | None = None) -> Any:
    self._id += 1
    req = {"jsonrpc": "2.0", "id": self._id, "method": method, "params": params or {}}
    self.proc.stdin.write(json.dumps(req) + "\n")
    self.proc.stdin.flush()
    line = self.proc.stdout.readline()
    resp = json.loads(line)
    if "error" in resp:
        raise RuntimeError(resp["error"])
    return resp["result"]

def _notify(self, method: str, params: dict | None = None) -> None:
    req = {"jsonrpc": "2.0", "method": method, "params": params or {}}  # 无 id
    self.proc.stdin.write(json.dumps(req) + "\n")
    self.proc.stdin.flush()
  4. ✅ 验证：

常见报错

  • 死循环 / 不停：确认无 tool_calls 即返回；max_turns 是否被误删；模型是否反复调同一失败工具（把失败原因写清楚喂回即可自我修复）。
  • 回填后模型「忘了」调过工具：assistant 消息没保留 tool_calls，或 tool 消息缺 tool_call_id（loop.py 已处理，别改坏）。
  • bash 卡住：命令在等交互输入（stdin）；靠 timeout 兜底，并在工具描述里声明不支持交互命令。
  • 工具重名 报错：build_default_registry 里重复注册了同名工具，检查是否粘贴了两遍。
  • FakeBackend 下完不成真任务：正常，export DEEPSEEK_API_KEY 后再跑步骤 8。
  • edit 总是匹配失败 / 报 old 不唯一：old 的空格/缩进要和文件完全一致；先 read（带行号）再照抄原文片段做 old；不唯一就把 old 扩大到包含上下文，直到只匹配一处。
  • grep 报 rg not found：装 ripgrep；或临时改用 subprocess 调 grep -rn 兜底。
  • token 爆 / 很慢：grep/read 结果没截断；把 max_lines/max_bytes/max_items 调小。
  • compaction 后模型「失忆」：摘要漏了任务目标/关键发现——摘要 prompt 明确保留三样（目标/发现/已完成），且最近 K 轮不要压缩。
  • compaction 太频繁：budget 阈值设太低——调高 budget，让它只在接近上限时触发。
  • web_fetch 结果撑爆上下文：没截断或 max_tokens 设太大——确认返回值走了 truncate_observation。
  • ModuleNotFoundError: markdownify（或 httpx）：依赖未装——pip install httpx markdownify。
  • MCP 握手卡住 / 读不到响应：没 flush，或 server 把日志打到了 stdout 污染 JSON → 确认每次写完都 flush()，server 日志走 stderr。
  • notifications/initialized 后一直等 result：通知本就无响应 → 用 _notify（不读 stdout），别当成 _rpc。
  • json.loads 读到半条消息：一行未读全 → 约定一行一条（\n 分隔）、Popen 用 bufsize=1；echo_server 已这么做。
  • filesystem server 起不来：无 Node/npx → 先用 echo 或自写 calc_server 完成本日目标，filesystem 作加分。
