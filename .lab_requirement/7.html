Day 7 下午 · Agent 记忆系统：持久化 + 召回注入 + remember 工具

一句话目标：为 agent 补上一层跨会话记忆——关掉会话再打开，它仍记得你的项目约定。 本次产出：一个 Memory（write 落盘 + recall 读回）、会话开始时召回注入 system prompt、一个 remember 写入工具、结构化记忆的更新与遗忘、一份本组项目的 MEMORY.md（CLAUDE.md 式）。 涉及的 starter 文件：新建 agent/memory.py；注入点在 agent/cli.py / agent/prompts.py；工具注册在 tools/。（本课新增内容，starter 暂无 TODO[Day7] 桩，按下面步骤新建即可。）

开始之前

  • 所有命令都在 starter/ 目录下运行：cd starter。
  • 承接前几天：agent 已有主循环、工具集、MCP、Skills、安全层。
  • 先想清楚边界（讲义 §2）：今天做的是跨会话持久状态，不是 Day 4 的上下文压缩，也不是 RAG 检索。

步骤 1 · 最小记忆：持久化 + 召回（新建 agent/memory.py）

  1. 记忆的本质：把"跨会话仍成立的信息"写到会话之外的文件，下次会话读得回（讲义 §6）。先做最小版——纯文本项目记忆： 
    from pathlib import Path

class Memory:
    def __init__(self, path="MEMORY.md"):
        self.path = Path(path)

    def write(self, note: str):
        """写入一条记忆（追加落盘 = 持久化）。"""
        with open(self.path, "a", encoding="utf-8") as f:
            f.write("- " + note.strip() + "\n")

    def recall(self, query: str = "") -> str:
        """召回：最简版本 = 读回全部（策略 A）。"""
        return self.path.read_text(encoding="utf-8") if self.path.exists() else ""
  2. ✅ 验证：跨"会话"持久（关键！）——写一次，新建实例再读，仍在： 
    python -c "from agent.memory import Memory; m=Memory('MEMORY.md'); m.write('本项目时间戳一律用 UTC')"
python -c "from agent.memory import Memory; print(Memory('MEMORY.md').recall())"   # 应打印刚写的那条

步骤 2 · 召回注入 system prompt（文件：agent/cli.py 或 agent/prompts.py）

  1. 记忆只有回到上下文里模型才看得见——和 Day 5 注入 skill catalog 同一手法。会话开始时召回、拼进 system prompt： 
    from agent.memory import Memory
mem = Memory("MEMORY.md")
recalled = mem.recall()
system = SYSTEM_PROMPT
if recalled.strip():
    system += "\n\n# 关于本项目 / 用户的已知记忆（相关时遵循）\n" + recalled
agent = AgentLoop(backend, reg, system)
  2. ✅ 验证：先写一条约定，再跑一个相关任务，看模型是否遵循： 
    python -c "from agent.memory import Memory; Memory('MEMORY.md').write('包管理器用 pnpm，不要用 npm')"
python -m agent.cli "给项目加一条安装依赖的命令示例"     # 期望它用 pnpm 而非 npm

步骤 3 · remember 工具：让模型自己写记忆（文件：tools/）

  1. 把写入做成一个工具，模型判断"这值得长期记"时自己调用（讲义 §4.1）： 
    def _remember(note: str) -> str:
    Memory("MEMORY.md").write(note)
    return "已记住：" + note

remember_tool = Tool(
    name="remember",
    description="当用户告诉你一条应长期记住的项目约定 / 偏好 / 关键决策时，调用它写入持久记忆。",
    parameters={"type": "object",
                "properties": {"note": {"type": "string"}},
                "required": ["note"]},
    run=lambda note: _remember(note))
    把它注册进 registry（和其它工具一样）。
  2. ✅ 验证：让 agent 记一条，再开新会话确认它记得： 
    python -m agent.cli "记住：本项目所有 API 返回时间戳用 ISO-8601"   # 应调用 remember 工具
python -m agent.cli "我们项目时间戳用什么格式？"                    # 新会话，应答 ISO-8601

步骤 4 · 更新与遗忘：结构化记忆（进阶，agent/memory.py）

  1. 纯文本只增不改会陈旧、膨胀（讲义 §4.3/§4.4、误区二）。加一个 KV 结构化记忆，支持按 key 覆盖与删除： 
    import json
class KVMemory:
    def __init__(self, path="memory.json"):
        self.path = Path(path)
        self.data = json.loads(self.path.read_text()) if self.path.exists() else {}
    def _save(self):
        self.path.write_text(json.dumps(self.data, ensure_ascii=False, indent=2))
    def remember(self, key, value): self.data[key] = value; self._save()   # 写入/更新
    def forget(self, key):          self.data.pop(key, None); self._save()  # 遗忘
  2. ✅ 验证：把 package_manager 从 npm 改成 pnpm（覆盖），再删掉一条临时记忆，确认落盘正确： 
    python -c "from agent.memory import KVMemory; k=KVMemory(); k.remember('pm','npm'); k.remember('pm','pnpm'); print(k.data)"

步骤 5 · 写一份本组项目记忆 MEMORY.md（CLAUDE.md 式）

  1. 按讲义 §3.3，为本组项目写一份真实的项目记忆：技术栈、约定、常用命令、踩过的坑。判据是"跨会话仍成立、且会影响未来行为"（误区一：别记闲聊）。 
    # 项目记忆
## 约定
- 时间戳一律用 UTC / ISO-8601
- 包管理器：pnpm；缩进：4 空格
## 常用命令
- 测试：pnpm test
## 踩过的坑
- 认证已从 session 迁到 JWT，别再用 session
  2. ⚠️ 脱敏（呼应 Day 6）：记忆会被持久化、反复注入、可能随仓库提交——不要写密钥 / 密码 / 隐私。
  3. ✅ 验证：关掉所有会话，重开，跑一个需要用到这些约定的任务，确认 agent 遵循。

常见报错

  • 现象：新会话读不到上次写的记忆 → 写入没落盘，或读的是不同路径 → 确认 write 真的 open(...,"a") 到同一个 path。
  • 现象：模型无视记忆 → 召回内容没进 system prompt，或写得太泛 → 确认步骤 2 注入生效；记忆要具体（"用 pnpm"而非"注意包管理"）。
  • 现象：记忆越攒越乱、召回噪声大 → 什么都记了（误区一）→ 只记"跨会话成立且影响行为"的；引入更新 / 遗忘。
  • 现象：remember 工具没被调用 → description 没说清"何时用" → 参考 skill description 的写法（Day 5），写明触发条件。
  • 现象：把整个代码库塞进"记忆" → 记忆 / RAG 错配（误区三）→ 外部只读大语料走 RAG（Day 4），自己攒的可变状态才是记忆。
